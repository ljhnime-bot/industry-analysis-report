# Industry Analysis (산업 분석)

투자 리서치 관점에서 산업·시장·섹터를 입체적으로 분석하기 위한 스킬이다. 시장의 돈의 흐름,
규제 환경, 기술 트렌드, 그리고 대상의 성장성·경쟁우위라는 4개 축을 데이터로 검증하고
하나의 투자 논리(thesis)로 종합한다.

핵심 원칙: **분석은 의견이 아니라 증거다.** 모든 주장에는 출처와 수치가 따라야 하며,
숫자 하나는 반드시 둘 이상의 출처로 교차검증(triangulation)한다.

---

## 0. 시작 전 — 스코프 합의

분석을 시작하기 전에 사용자와 다음을 1분 안에 정렬한다. 이미 프롬프트에 답이 있으면 다시 묻지 말고 진행한다.

- **분석 대상**: 산업 전체 / 특정 세그먼트 / 특정 기업 / 특정 기술 중 무엇인가?
- **목적**: 투자 검토 / 시장 진입 / 경쟁 벤치마킹 / 단순 학습 — 목적에 따라 깊이와 톤이 달라진다.
- **지역**: 글로벌 / 한국 / 특정 국가. (데이터 출처와 규제 기관이 완전히 달라짐)
- **깊이**: 빠른 개요(30분 수준) / 심층 리포트(완전 분석).

목적이 모호하면 한 가지만 묻고, 나머지는 합리적으로 가정한 뒤 가정을 명시한다.

---

## 1. 4대 핵심 분석 영역

산업 분석은 아래 4개 영역을 모두 다뤄야 입체적인 그림이 완성된다. 각 영역은 유기적으로 연결되어 있다.

### ① M&A 및 투자 동향 — "돈이 어디로 흐르는가"
- 어떤 섹터·기술에 자금이 집중되는가, 딜 규모와 빈도는?
- 밸류에이션이 역사적으로 높은 수준인가 낮은 수준인가? (멀티플 비교)
- 주요 인수자는 누구이며 그들의 전략은? (전략적 투자자 vs 재무적 투자자)
- → 우리가 검토하는 투자가 시장 흐름에 순응하는지 역행하는지 판단하는 기준점.

### ② 정책 및 규제 동향 — "게임의 룰"
- 사업의 근간을 흔들 수 있는 규제·법안은? 반대로 성장 촉매가 되는 정책은?
- 기업결합 심사, 산업 육성 정책, 환경·데이터 규제, 인허가 장벽.
- → 리스크 회피(수비)이자 규제 변화 속 기회 포착(공격)의 출발점.

### ③ 산업 및 기술 트렌드 — "미래 가치와 방향성"
- 현재 주도 기술이 3~5년 뒤에도 유효한가? 파괴적 기술(예: AI)이 지형을 어떻게 바꾸는가?
- 기술 수용 주기(S-curve) 상 현재 위치는? 소비자 행동·가치관 변화는?
- → 과거가 아닌 미래에 베팅하기 위한 근거.

### ④ 성장성 및 경쟁우위 분석 — "그래서 돈을 버는가"
- **정량**: 시장 성장률(CAGR), 매출·이익 증가세, 침투율, 단위경제성.
- **정성**: 지속 가능한 경쟁우위(Moat) — 독점 기술, 브랜드, 네트워크 효과, 전환비용, 규모의 경제.
- → 투자의 성패를 가르는 최종 관문이자 본질.

## 2. 표준 워크플로우

순서대로 진행하되, 빠른 개요 요청이면 4~6단계를 압축한다.

1. **스코프 합의** (§0)
2. **시장 정의·규모화**: 대상 시장을 명확히 정의하고 TAM/SAM/SOM과 CAGR을 추정한다.
   → 시장 규모 수치는 **반드시 2개 이상 출처로 교차검증**한다 (§4 삼각검증 필수).
3. **4대 영역 리서치** (§1): 각 영역별로 §3의 출처에서 근거를 수집한다.
   각 영역은 분리해서 검색한다 — 한 번의 뭉뚱그린 검색은 표면적 결과만 준다.
4. **프레임워크 적용**: `references/frameworks.md`의 도구로 구조화한다
   (Porter's 5 Forces, PESTEL, 가치사슬, 경쟁 포지셔닝, 기술 수용 주기 등).
5. **종합·논리화**: 흩어진 사실을 하나의 thesis로 묶는다 — "이 산업/기업은 왜, 어떻게 돈을 벌며,
   무엇이 그것을 지속시키고, 무엇이 그것을 위협하는가."
6. **리스크·반대논리**: 강세 논리만 쌓지 말고 약세 시나리오와 핵심 리스크를 명시한다.
7. **리포트 작성**: `assets/report-template.md` 구조를 따른다.

## 3. 정보 출처와 접근 방법

출처는 (A) Claude가 직접 접근 가능, (B) API/MCP로 호출 가능, (C) 유료·로그인 필요 → 사용자 협조 필요
세 등급으로 나뉜다. **전체 사이트별 상세 접근 지침과 사용자 행동 가이드는
`references/source-access.md`를 반드시 참조하라.** 아래는 빠른 요약이다.

### Claude가 직접 접근 가능 (web_search → web_fetch)
- **정부·규제 기관**: FTC, EU Commission, 공정거래위원회, 기획재정부·산업통상자원부, ETRI — 전면 공개.
- **공시**: DART/OpenDART(한국, 이 환경에 MCP 보유), SEC EDGAR(미국), 기업 IR 자료.
- **공공 통계 API**: 통계청 KOSIS, 한국은행 ECOS, World Bank, OECD, IMF.
- **뉴스·2차 시장조사**: TechCrunch, 주요 경제지, 그리고 Mordor·MarketsandMarkets·Grand View 등
  2차 시장조사 기관의 요약 페이지 (Statista 본문보다 오히려 검색에 잘 잡힌다).
- **FnGuide 무료 포털**: comp.wisereport.co.kr / comp.fnguide.com (국내 상장사 재무·컨센서스 일부).

### API/MCP로 호출 가능
- **네이버 데이터랩**: 이 환경에 `NaverSearch-datalab_*` MCP 도구 보유 → 검색어 트렌드·쇼핑 트렌드 직접 호출.
  (공식 데이터랩 API도 무료 키 발급 가능)
- **DART(OpenDART)**: 무료 인증키로 재무제표·공시 호출 (이 환경에 `opendart-*` MCP 일부 보유).
- **Crunchbase / Statista / S&P 등**: 유료 API 존재하나 키·구독 필요 → §C로 분류.

### 유료·로그인 필요 → 사용자 협조 필요 (직접 접근 불가)
- **전면 구독제(직접 접근 거의 불가)**: PitchBook, S&P Capital IQ, MergerMarket, CB Insights,
  Euromonitor(Passport), NICE BizLINE, Statista 프리미엄, Forrester 유료 리포트.
- **전문가 네트워크(ENS)**: GLG·AlphaSights류 — 사람 인터뷰 서비스라 Claude가 대행 불가.

이 경우 Claude의 행동 원칙 (상세는 `references/source-access.md`):
1. 먼저 **직접 접근 가능한 동급 대체 출처**로 최대한 채운다 (예: PitchBook 딜 데이터 →
   공개 M&A 뉴스 + 기업 공시 + Crunchbase 공개 프로필로 근사).
2. 그래도 핵심 공백이 남으면, 사용자에게 **구체적으로** 요청한다:
   "PitchBook에서 'X 섹터 2024~2025 딜 리스트(딜 규모·인수자·밸류에이션 멀티플 포함)'를
   추출해 붙여넣어 주시면 분석에 반영하겠습니다." — 막연히 "자료 주세요"가 아니라 항목을 특정한다.
3. 사용자가 API 키를 보유했다면 사용 방법을 안내하되, **키 값 자체는 채팅에 입력하지 말고**
   사용자가 직접 환경변수/도구 설정에 넣도록 안내한다 (보안).

## 4. 데이터 삼각검증 — 가장 중요한 규율

시장조사 수치는 출처마다 극단적으로 다르다. 실제로 동일한 "의료미용기기 시장 규모"가
출처에 따라 6.7십억 달러 ~ 32십억 달러로 **5배 가까이** 차이 났다. 따라서:

- **단일 출처 수치를 단정적으로 인용하지 마라.** 항상 범위로 제시하고 출처를 병기한다:
  "글로벌 의료미용기기 시장은 2025년 기준 약 170~210억 달러로 **추정**된다(출처에 따라 상이),
  CAGR 10~13% 수준으로 전망된다."
- **출처의 등급을 따진다**: 1차 출처(정부 통계, 기업 공시, 감사보고서) > 신뢰도 높은 2차
  (S&P, 학술지, 주요 컨설팅) > 일반 시장조사 보도자료 > 블로그·포럼.
  시장조사 보도자료는 정의(범위)와 방법론이 제각각이라 그대로 믿지 않는다.
- **정의를 확인한다**: "시장 규모"가 장비만인지 소모품 포함인지, 글로벌인지 특정 지역인지,
  매출 기준인지 출하 기준인지에 따라 숫자가 달라진다. 사과와 오렌지를 비교하지 마라.
- **상충하면 더 검색한다**: 두 출처가 크게 다르면 한쪽으로 단정하지 말고 추가 출처로 좁히거나,
  차이의 원인(정의·연도·방법론)을 밝힌다.

## 5. 산출물

- 빠른 개요 요청 → 대화형으로 간결하게(프로즈 중심), 핵심 수치·논리·리스크만.
- 심층 리포트 요청 → `assets/report-template.md` 구조의 문서 파일로 작성.
- 시장 규모·점유율·성장률 비교처럼 **데이터의 형태**를 보여줄 때는 표나 차트를 활용한다.
- 모든 핵심 주장에 출처를 단다. 추정치는 "추정"임을 명시하고 가정을 드러낸다.
- **환각 금지**: 모르는 수치를 지어내지 않는다. 못 찾으면 "공개 자료로는 확인되지 않음"이라고 적고,
  필요한 경우 사용자에게 유료 출처에서 가져올 항목을 특정해 요청한다.
