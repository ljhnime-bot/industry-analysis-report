# 정보 출처 접근 가이드 (Source Access Guide)

산업 분석에 쓰이는 주요 출처별로, Claude가 (A) 직접 접근 가능한지, (B) API/MCP로 호출 가능한지,
(C) 유료·로그인이라 사용자 협조가 필요한지를 정리한다. 등급은 일반적 접근 모델 기준이며,
사이트 정책은 바뀔 수 있으므로 실제 시도 후 결과로 판단한다.

접근 메커니즘 메모:
- Claude의 `web_fetch`는 **검색 결과·이전 페치에 등장한 URL만** 가져올 수 있다. 따라서 직접 접근도
  "먼저 `web_search` → 결과 URL을 `web_fetch`" 순서를 따른다. 임의 URL을 바로 페치할 수 없다.
- 유료 사이트는 검색 스니펫은 보여도 본문은 로그인 벽에 막혀 페치가 실패하거나 빈 내용이 온다.
- 흥미로운 현실: Statista 같은 곳은 본문이 검색에 거의 안 잡히는 반면, 동급 데이터를 제공하는
  2차 시장조사 기관(Mordor, MarketsandMarkets 등)은 요약이 잘 노출된다. 즉 "그 사이트"가 아니라
  "그 데이터"를 찾는 쪽이 효율적이다.

## Claude가 추가로 직접 활용할 수 있는 출처 (원문에 없던 보강)

직접 접근 가능하면서 산업 분석에 강력한 1차 출처들. 유료 플랫폼의 공백을 상당 부분 메운다.

- **DART / OpenDART** (한국 공시): 사업보고서·감사보고서·재무제표. 무료 인증키. 이 환경에 `opendart-*` MCP 일부 보유.
- **SEC EDGAR** (미국 공시): 10-K, 10-Q, S-1, 8-K. 전면 공개. 산업·경쟁사 분석의 1차 출처.
- **통계청 KOSIS / 한국은행 ECOS**: 거시·산업 통계. 무료 공개 API.
- **World Bank / OECD / IMF Data**: 글로벌 거시·산업 데이터. 무료 공개 API.
- **기업 IR 페이지·실적발표 자료**: 세그먼트 매출, 가이던스, 시장 점유율 주장 (단, 자사 유리하게 편향).
- **증권사 리서치 리포트**: 일부 공개. 컨센서스·목표주가·산업 전망. (FnGuide·한경컨센서스 등에서 일부 노출)
- **특허**: KIPRIS(한국), Google Patents, USPTO — 기술 트렌드·경쟁사 R&D 방향.
- **arXiv·학술 검색**: 딥테크 산업의 기술 성숙도 판단.
- **2차 시장조사 기관**: Mordor Intelligence, MarketsandMarkets, Grand View Research,
  Fortune Business Insights, Technavio, Persistence — 시장 규모·CAGR 요약이 검색에 잘 노출.
  단, 정의·방법론이 제각각이므로 **반드시 복수 출처로 범위 제시**(SKILL.md §4 삼각검증).

## 유료 출처를 만났을 때의 행동 절차

핵심 데이터가 유료(C 등급) 뒤에 있을 때, 순서대로 처리한다.

### 1단계 — 대체 출처로 최대한 채운다
유료 플랫폼의 데이터는 대개 공개 출처 조합으로 근사할 수 있다:

| 원하는 데이터(유료 출처) | 직접 접근 대체 경로 |
|---|---|
| PitchBook/MergerMarket 딜 리스트 | 공개 M&A 뉴스 + 기업 공시(8-K/주요사항보고) + Crunchbase 공개 프로필 |
| S&P Capital IQ 재무·멀티플 | DART/SEC EDGAR 원문 재무 + FnGuide 무료 포털 + 증권사 컨센서스 |
| Statista/Euromonitor 시장 규모 | 2차 시장조사 기관 복수 + 정부 통계 + 상장사 IR의 시장 점유율 언급 |
| CB Insights 기술 맵 | TechCrunch + 특허 검색 + arXiv + 기업 블로그·기술 발표 |
| Euromonitor 소비자 데이터 | 통계청·트렌드모니터 무료분 + 네이버 데이터랩 MCP + 설문 보도자료 |

### 2단계 — 공백이 남으면, 항목을 특정해 사용자에게 요청
막연한 "자료 주세요"가 아니라, 무엇을 어떤 형식으로 줄지 콕 집는다. 예시:

> "PitchBook 접근 권한이 있으시면, **'[섹터]' 2023~2025년 딜 리스트**를 추출해 주세요 —
> 컬럼은 (딜 명 / 인수자 / 피인수자 / 딜 규모 / EV/EBITDA 또는 매출 멀티플 / 딜 단계).
> 표나 CSV로 붙여넣어 주시면 M&A 동향 분석에 그대로 반영하겠습니다."

> "Statista에서 보이는 **'[시장]' 연도별 시장 규모(2020~2030E)와 CAGR**, 그리고 그 수치의
> **출처·방법론 각주**를 함께 캡처/복사해 주세요. 정의가 분석의 정확도를 좌우합니다."

### 3단계 — API 키를 보유한 사용자 안내 (보안 우선)
사용자가 Crunchbase/Statista/S&P 등의 유료 API 키를 가진 경우:
- **키 값을 채팅창에 붙여넣지 말 것**을 먼저 안내한다. 채팅 입력은 노출 위험이 있다.
- 사용자가 직접 자신의 코드/도구 환경의 **환경변수나 커넥터 설정**에 키를 넣고, Claude는
  그 도구를 호출하는 방식으로 진행한다.
- 이 환경에서 즉시 가능한 무료 키 기반 출처는 **DART(OpenDART)**, **네이버 데이터랩/검색**,
  **공공데이터포털 API** 등이다 — 유료 글로벌 플랫폼보다 이쪽을 먼저 권한다.

### 4단계 — ENS(전문가 네트워크)
GLG·AlphaSights·Third Bridge류는 전·현직 실무자와의 유료 인터뷰 서비스다. Claude가 대행할 수 없다.
대신 Claude가 **인터뷰 질문지 설계**, 인터뷰 후 **노트 정리·교차분석**을 도울 수 있다. 사용자에게:
> "전문가 인터뷰는 직접 진행하셔야 합니다. 원하시면 정량 분석에서 드러난 **검증 필요 가설**을
> 바탕으로 인터뷰 질문지를 만들어 드릴게요. 인터뷰 노트를 주시면 분석에 통합하겠습니다."

## 빠른 의사결정 규칙

1. 규제·정책·공시·거시통계가 필요하다 → **A 직접**, 바로 검색+페치/공공 API.
2. 검색어·소비 트렌드가 필요하다 → **네이버 데이터랩 MCP** 우선, 보조로 검색.
3. 시장 규모·성장률이 필요하다 → 2차 시장조사 **복수** 검색 → 범위로 제시 + 삼각검증.
4. 비공개 딜·PE/VC 상세·프리미엄 시장 데이터가 필요하다 → 대체 출처 시도 → 공백은 항목 특정해 사용자 요청.
5. 살아있는 현장 목소리가 필요하다 → 사용자가 ENS 인터뷰 진행, Claude는 질문 설계·노트 분석 지원.
